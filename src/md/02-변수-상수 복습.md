# 02 변수와 상수
## 1. 변수 선언 (`let` 키워드 사용)
let 변수이름 = 값
```
 let initialApples = 4 /변수를 선언할 시 영어 사용 권장!
 ```
* let: 키워드
* initialApples :변수이름
* = :할당 연산자
* 4: 값

## 1) `변수 이름`
### 1️⃣ 변수 이름 규칙
  * 연결된 한 단어로 작성
  * `알파벳,숫자`,`_ 또는 $` 등의 기호로 구성되어야 함
  * 변수 이름이 `숫자로 시작되어선 ❌`
  * 변수 이름으로 `예약어 사용할 수 ❌`

  ### 2️⃣ 변수 이름 표기법
  - initialApple //camelCase
  - InitialApple //PascalCase
  - initial_apples //snake_case

  | 표기법 | 사용대상 |예시|
  |------|------|------|
  |camelCase|변수,함수|userName,gettTotalPrice|
  |PascalCase|클래스,생성자|User,ShoppingCount|
  |UPPER_SNAKE_CASE|상수|MAX_SIZE, API_KEY|
  |kebab-case|HTML,CSS|user-name, main-content|

## 2) 값
: **변수에 기억되는 값**으로, 기본형(문자열, 숫자 등)이거나
객체형(배열, 함수 등)일 수 있다.

:변수값을 할당하지 않을 경우 기본값은 **Undefined** 가 된다.

## 3) 할당(대입)연산자
: `자바스크립트에서는 등호( = )기호는 할당을 의미합니다.`
: 변수는 **새로운 값**을 다시 할당할 수 있음.
```
let count = 0
console.log(count) //0
//이후에 새로운 값 다시 할당

count = 5
console.log(count) //5
```

## 4) 변수 선언 특징
### 1️⃣ 변수값 `재할당` 가능
```
let truthy = true
truthy = false
console.log('truthy 변수에 재할당된 값:',truthy)
출력값: 선언 후 나중에 할당한 값: 라모스
```

### 2️⃣ 선언하고 `나중에 할당` 가능
```
let gamePlayer  //변수 선언
gamePlayer = '라오스'
console.log('선언 후 나중에 할당한 값:',gamePlayer)
출력값: 선언 후 나중에 할당한 값: 라오스
```

### 3️⃣ 여러 변수 `동시 선언` 가능
``` 
let screen_width, screen_height
```
### 4️⃣ `이후`에 선언된 변수에 `값` 할당
```
screen_width = 290
screen_height = 842
console.log('screen_height =', screen_height)
console.log('screen_height =', screen_height)
출력값: 
screen_width = 290
screen_height = 842

```

# 2. const 상수: `변하지 않는` 수
:JaVaScript 에서 변수를 선언할 때 사용되는 키워드는
let,var,const 이다. 그중 const 키워드는 상수를 선언할 때 사용된다.

* 상수의 규칙
1. 상수 선언과 `동시`에 값 할당 (필수 !)
```
const 상수 = '고정된 값'
console.log(상수)
```
2. 상수는 값을 `재할당 ❌`
3. 여러 `상수 동시 선언`가능
```
const MIN_VALUE = 10, MAX_VALUE = 100
console.log(MIN_VALUE,  MAX_VALUE )
출력값: 10, 100
```

---
#요약- 변수를 사용하면 의미있는 코드를 작성할 수 있습니다.
- 변수는 데이터 값을 기억하는 데 사용됩니다.
- 등호(`=`) 기호는 할당(⚠️ 같음을 의미하지 않음)을 의미합니다.
- 변수 이름은 일반적으로 카멜케이스(camelCase) 표기 방법으로 사용됩니다.
- 변수 이름에 예약어(Reserved Words)는 사용될 수 없습니다.
- 변수는 `let`으로 선언하고, 상수는 `const`로 선언합니다.
- `var` 키워드는 더 이상 사용하지 않는 것이 좋습니다.
